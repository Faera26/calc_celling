# Supabase Structure

Каноническая история миграций теперь живёт в `infra/supabase/migrations/`.

Что это значит:
- новые изменения схемы добавляем только сюда;
- порядок файлов отражает эволюцию базы;
- версии `20260420063912`, `20260420070147`, `20260420072018`, `20260420073728` сохранены специально, потому что они уже есть в удалённой истории Supabase;
- финальные доработки по snapshot и hardening тоже оформляем отдельными миграциями, а не ручными SQL-файлами вне этой папки.

Практика:
- миграции должны быть идемпотентными там, где это возможно;
- одноразовые диагностики, временные фиксы и ручные проверки не считаем миграциями;
- при изменении схемы сначала обновляем миграцию здесь, затем применяем изменение в Supabase.
